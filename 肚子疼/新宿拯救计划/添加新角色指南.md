# 新宿拯救计划：添加新角色指南

本指南将指导你如何为 `整合版-MVU.html` 添加新的角色卡片。通过重构后的代码，你只需要完成以下三个步骤即可实现新角色的完整集成。

---

## 步骤 1：在 `CHARACTER_CONFIG` 中添加配置

在 HTML 文件的 `<script>` 部分找到 `CHARACTER_CONFIG` 对象，并仿照现有角色添加新条目。

### 配置模板：
```javascript
const CHARACTER_CONFIG = {
    // ... 现有角色
    NEW_CHAR_ID: { // 内部唯一 ID，例如 'shiori'
        id: 'shiori',
        name: '新名', // 简称，用于回复显示
        fullName: '全名', // 显示在卡片上的全名
        nameEn: 'ENGLISH NAME', // 英文名（用于卡片背景装饰）
        avatar: '头像 URL', // 圆形头像
        hero: '默认立绘 URL', // 租借页面展示的大立绘
        heroOutside: '表立绘 URL', // 未确立关系时展示的立绘（详情页）
        heroInside: '里立绘 URL', // 确立关系后展示的立绘（详情页）
        themeClass: 'theme-shiori', // CSS 主题类名
        styleClass: 'style-shiori', // 聊天气泡样式类名
        color: '#ff6b6b', // 代表颜色（用于进度条、标签背景）
        rank: 'TOP X', // 排名文字
        rankIcon: 'fas fa-star', // 排名图标
        tags: ['标签1', '标签2'], // 角色标签
        varPrefix: '新角色变量前缀', // 必须与变量结构.js中的前缀一致
        rentInfo: { // 租借页面的详细信息
            tagline: "一句话简介",
            rate: "¥12,000",
            rating: "5.0 ★",
            age: "20",
            type: "性格类型",
            bio: "个人独白文字...",
            map: "活动范围描述...",
            mapTags: ["#标签1", "#标签2"],
            rules: "规则与红线描述...",
            schedule: "档期安排描述..."
        }
    }
};
```

## 步骤 2：添加 CSS 样式

在 `<style>` 标签中为新角色添加主题和聊天样式。

### 1. 主题颜色与背景（Mind Scan 使用）
```css
.theme-shiori { --theme-color: #ff6b6b; }
.theme-shiori .face-inner { background: linear-gradient(135deg, #fff5f5 0%, #ffe0e0 100%); }
.theme-shiori .fill-mind { background: #ff6b6b; }
.theme-shiori .val { color: #ff6b6b; }
```

### 2. 聊天气泡样式
```css
.style-shiori .msg-avatar { border: 2px solid #ff6b6b; }
.style-shiori .msg-name { color: #ff6b6b; }
.style-shiori .msg-bubble { 
    border-radius: 4px 18px 18px 18px; 
    background: linear-gradient(145deg, #fff0f0, #ffe5e5); 
    color: #333; 
    border: 1px solid rgba(255, 107, 107, 0.3); 
}
```

### 3. 立绘缩放与适配（详情页使用）
不同角色的立绘尺寸不同，推荐使用 CSS 变量进行微调（不会与呼吸动画冲突）：
```css
/* 默认状态（表立绘） */
.page-shiori .char-img { 
    height: 100%;           /* 基础高度 */
    --off-x: 5%;           /* 水平位移：正数向右，负数向左 */
    --off-y: 0px;          /* 垂直位移：正数向下，负数向上 */
    --scale: 1;            /* 缩放比例 */
    --rotate: 0deg;        /* 旋转角度 */
}

/* 确立关系状态（里立绘） */
.page-shiori.is-in-love .char-img { 
    height: 95%; 
    --off-x: 0%; 
    --scale: 0.95;
    --rotate: 2deg;
}
```

## 步骤 3：在 HTML 中添加详情页（Overlay）

由于每个角色的详情页立绘布局可能不同，你需要手动在 `<!-- ========== OVERLAYS: HOME DETAILS (INSIDE) ========== -->` 区域添加对应的详情页 HTML。

### 详情页模板：
```html
<div class="page-detail page-NEW_CHAR_ID" id="page-NEW_CHAR_ID">
    <!-- 顶部导航 -->
    <div class="detail-top-nav">
        <i class="fas fa-chevron-left nav-back" onclick="closePageDetail('NEW_CHAR_ID')"></i>
        <div class="nav-title">TomorrowsGirlfriend</div>
        <i class="fas fa-ellipsis-h"></i>
    </div>
    
    <!-- 背景大文字装饰 -->
    <div class="big-typo-bg"><span>NAME</span><span class="typo-stroke">03</span></div>
    
    <!-- 立绘容器（src 会被 JS 根据恋爱状态动态替换） -->
    <div class="char-container"><img src="默认表立绘 URL" class="char-img"></div>
    
    <!-- 状态面板（好感度/信赖度） -->
    <div class="status-wrapper">
        <div class="status-panel">
            <div class="stat-item">
                <div class="stat-head"><span>好感度</span> <span id="NEW_CHAR_ID-detail-love-text">0/400</span></div>
                <div class="progress-box"><div class="progress-fill" id="NEW_CHAR_ID-detail-love-bar"></div></div>
            </div>
            <!-- ... 信赖度以此类推 ... -->
            <div class="icon-row">
                <!-- 联系方式和恋爱状态图标 -->
                <div class="icon-btn">
                    <div class="icon-circle" id="NEW_CHAR_ID-contact-icon"><i class="fas fa-lock"></i></div>
                    <span id="NEW_CHAR_ID-contact-text">未交换</span>
                </div>
            </div>
        </div>
    </div>
    
    <!-- 其他装饰性 Sticker 建议直接复用现有结构，只需修改 ID 前缀即可 -->
</div>
```

---

## 💡 注意事项

1. **变量对应**：确保 `CHARACTER_CONFIG` 中的 `varPrefix` 与 `变量结构.js` 中 `stat_data` 下的键名完全一致。
2. **立绘切换逻辑**：系统会自动检测变量中的 `恋爱状态`。
    - 如果为 `true`，详情页容器会添加 `.is-in-love` 类，并切换图片为 `heroInside`。
    - 如果为 `false`，则移除该类，并切换图片为 `heroOutside`。
3. **ID 唯一性**：新角色的 ID（如 `shiori`）在整个 HTML 文件中应保持唯一，用于 DOM 元素的 ID 绑定（如 `shiori-love-text`）。
4. **资源链接**：建议使用稳定的图床链接，避免立绘或头像失效。

完成以上步骤后，刷新页面，新角色将自动出现在主页列表、心声扫描页和租借页面中！
