# 什么叫请神上身请到了色孽？

## 项目结构

```
什么叫请神上身请到了色孽？/
├── 需求确认文档.md          # 需求收集和设计决策记录
├── 世界书条目配置表.yaml    # 世界书条目配置
├── 世界书/                   # 世界书条目文件（.md）
├── 脚本/                     # JavaScript脚本（如变量结构.js）
├── 正则/                     # 正则表达式文件（.json）
├── 开局/                     # 开局消息文件（.md）
└── 最终角色卡.json          # 打包后的角色卡（由脚本生成）
```

## 使用流程

### 1. 填写需求文档
编辑 `需求确认文档.md`，明确角色卡的各项需求。

### 2. 创建世界书条目
在 `世界书/` 目录下创建条目文件，并在 `世界书条目配置表.yaml` 中配置。

### 3. 配置变量系统（可选）
- 在 `脚本/` 目录下创建 `变量结构.js`（Zod Schema）
- 在 `世界书/` 目录下创建变量相关条目

### 4. 编写开局
在 `开局/` 目录下创建开局文件（如 `开局1.md`）。

### 5. 打包角色卡
使用打包脚本生成最终角色卡：

```bash
# 自动模式（从配置文件读取元数据）
python path/to/package_card_cli.py "什么叫请神上身请到了色孽？"

# 手动模式（命令行指定参数）
python path/to/package_card_cli.py "什么叫请神上身请到了色孽？" "世界书名" "角色名" "角色描述"
```

## 注意事项

1. 所有世界书条目文件必须放在 `世界书/` 目录下
2. 条目文件名必须与配置表中的 `文件名` 字段一致
3. 正则文件已预设，可根据需要修改
4. 变量模板文件已包含在 `世界书/` 目录中
