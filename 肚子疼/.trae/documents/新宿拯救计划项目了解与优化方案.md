## 优化建议
为了让你的项目更符合最新的酒馆助手开发规范，我建议进行以下调整：

### 1. 变量获取方式升级
目前的 HTML 中使用的是 `getAllVariables()`，建议改为 MVU 推荐的 `Mvu.getMvuData()` 接口，这样可以更精确地锁定消息楼层的数据。

### 2. 增强初始化安全性
在脚本开始运行前，添加对 `Mvu` 对象和 `stat_data` 数据的等待逻辑，防止在酒馆环境未准备好时发生报错。

### 3. 尺寸单位优化
HTML 中使用了部分 `vh` 单位（如人物立绘高度），这在酒馆的 iframe 环境中可能会导致显示异常。建议统一使用 `width` 配合 `aspect-ratio` 或者百分比单位。

### 4. 结构模块化
如果你希望利用模板的打包功能，我们可以将 HTML 里的 CSS 和 JS 抽离出来，放入 `src` 目录下的 `index.ts` 中，让 `index.html` 保持纯净。
